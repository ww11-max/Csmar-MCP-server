# CSMAR MCP服务器 - 快速开始指南

## 🚀 立即开始使用

### 第一步：重启Claude Code
**必须重启Claude Code** 以加载新的MCP服务器配置。

### 第二步：验证安装
在Claude Code中输入以下命令验证安装：

```python
# 验证MCP服务器是否加载
mcp__csmar__csmar_list_databases()
```

如果看到数据库列表，说明安装成功！

---

## 📊 基础使用示例

### 1. 探索可用数据
```python
# 列出所有可用数据库
mcp__csmar__csmar_list_databases()

# 查看"财务报表"数据库中的表
mcp__csmar__csmar_list_tables(database_name="财务报表")

# 查看"FS_Combas"表的字段
mcp__csmar__csmar_list_fields(table_name="FS_Combas")

# 预览表数据（前几行）
mcp__csmar__csmar_preview(table_name="FS_Combas")
```

### 2. 数据查询示例
```python
# 查询财务报表数据
mcp__csmar__csmar_query(
    table_name="FS_Combas",
    columns=["Stkcd", "ShortName", "Accper", "Typrep", "A001000000"],
    condition="Stkcd like '3%' and Typrep='A'",
    start_time="2020-01-01",
    end_time="2021-12-31",
    limit=5
)

# 查询记录数量
mcp__csmar__csmar_query_count(
    table_name="FS_Combas",
    condition="Stkcd like '3%'",
    start_time="2020-01-01",
    end_time="2021-12-31"
)
```

### 3. 专用工具
```python
# 获取股票数据
mcp__csmar__get_stock_data(
    stock_code="000001",        # 股票代码
    start_date="2023-01-01",    # 开始日期
    end_date="2023-12-31",      # 结束日期
    frequency="daily"           # 频率: daily, weekly, monthly
)

# 获取公司信息
mcp__csmar__get_company_info(stock_code="000001")

# 获取财务数据（需要配置API密钥）
mcp__csmar__get_financial_data(
    stock_code="000001",
    start_date="2020-01-01",
    end_date="2022-12-31",
    indicators=["ROE", "ROA", "EPS"]  # 财务指标
)
```

## ⚠️ 注意事项

### 查询限制
- **每次最多20万条记录**: 超过需要分页查询
- **相同条件30分钟限流**: 避免频繁查询相同条件
- **时间格式**: 必须使用 "YYYY-MM-DD" 格式

### 分页查询示例
```python
# 大数据集分页查询
条件 = "Stkcd like '3%' limit 0,200000"  # 第1页
# 条件 = "Stkcd like '3%' limit 200000,200000"  # 第2页

mcp__csmar__csmar_query(
    table_name="FS_Combas",
    columns=["Stkcd", "ShortName", "Accper", "Typrep"],
    condition=条件
)
```

## 🔧 故障排除

### 问题1: "MCP服务器未响应"
**解决方案:**
1. 确认已重启Claude Code
2. 检查`.mcp.json`文件路径
3. 手动测试Python客户端:
   ```bash
   cd /path/to/csmar-mcp-server
   echo '{"action":"list_databases","params":{}}' | python src/python_client.py
   ```

### 问题2: "数据库不存在"
**解决方案:**
1. 使用`csmar_list_databases()`获取准确名称
2. 检查数据库名称是否包含空格或特殊字符
3. 确认账号有该数据库访问权限

### 问题3: 查询结果为空
**解决方案:**
1. 检查时间范围是否正确
2. 验证查询条件语法
3. 使用`preview()`先查看数据格式
